About Bus Management System

A full-stack web application for OSSD Final Semester Project – UMT Y9

📌 Problem Statement

Many transport companies, schools, and colleges manage bus records manually. This leads to scheduling issues, booking errors, difficulty tracking passengers, and inefficient record management. This system automates all these operations.

🎯 Project Objective

Develop a web-based Bus Management System that allows administrators to manage buses, passengers, and bookings through a centralized, easy-to-use interface connected to a cloud PostgreSQL database.

👥 Target Users

  • 🔑 Administrator – Full system access
  • 🚌 Bus Manager – Manage buses and schedules
  • 🎫 Passengers – Book and view tickets

⚙️ Technology Stack

FastAPI Python PostgreSQL SQLAlchemy Supabase HTML5 CSS3 JavaScript Render Vercel GitHub

📡 API Endpoints (16 total)

  • POST /register  |  POST /login
  • POST /buses  |  GET /buses
  • GET /buses/{id}  |  PUT /buses/{id}  |  DELETE /buses/{id}
  • POST /passengers  |  GET /passengers
  • GET /passengers/{id}  |  PUT /passengers/{id}  |  DELETE /passengers/{id}
  • POST /bookings  |  GET /bookings  |  DELETE /bookings/{id}
  • GET /dashboard

🗄️ Database Schema

  • users – user_id, username, email, password
  • buses – bus_id, bus_number, bus_name, capacity, status
  • passengers – passenger_id, name, phone, email
  • bookings – booking_id, passenger_id, bus_id, seat_number, booking_date

🚀 Deployment

  • 🌐 Frontend → Vercel
  • ⚙️ Backend → Render
  • 🗄️ Database → Supabase (PostgreSQL)
  • 📦 Version Control → GitHub
